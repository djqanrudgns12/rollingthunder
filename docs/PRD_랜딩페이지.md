## 1. 배경 & 문제 정의 (As-Is)

현재 루트 경로의 상태:

| 문제 | 근거 |
|------|------|
| 루트(`/`)가 곧바로 로그인/회원가입 폼 | `src/app/page.tsx` — 서비스가 뭔지 모르는 신규 방문자에게 계정 생성부터 요구 |
| 서비스 소개·스크린샷·기능 설명 전무 | 첫 문장이 "Rolling Thunder 세션을 불러옵니다." 뿐 |
| **이용약관/개인정보처리방침 부재** | 실명(`name`)·닉네임 등 개인정보를 수집하면서 처리방침 미공개 → 개인정보 보호법 제30조(처리방침 수립·공개 의무) 리스크 |

이미 갖춰진 자산 (재활용 대상):

- **인증 흐름**: username + 가상 도메인(`@rt.local`) 기반 Supabase Auth, `keep_logged_in` 쿠키(1년), 게스트 → 회원 데이터 승계(`guestChips`/`guestInventory`/`guestEquipped`)
- **미들웨어 리다이렉트**: 로그인 유저가 `/` 접근 시 `/dashboard`로 이동 (`src/middleware.ts`) — **그대로 유지**
- **계정 완전 삭제 기능**: `deleteAccount()` — Admin API로 `auth.users` 삭제 + CASCADE (처리방침 "파기 절차"의 실체)
- **디자인 시스템**: CSS 변수 테마(`--bg-primary`, `--accent-primary`…), `glass-panel-heavy`, 네온 글로우, Outfit/Pretendard 폰트
- **차분 모드(calmMode)**: 2-tier 모션 저감 — 랜딩 애니메이션도 이를 존중해야 함

## 4. 랜딩 페이지 상세 스펙

### 4.1 히어로 섹션 — "중력이 결정하게 하라"

첫 화면에서 서비스의 본질(물리 엔진이 만드는 공정하고 극적인 추첨)을 **말이 아니라 체험**으로 전달한다.

**핵심 연출 (창의 요소):**

1. **인터랙티브 물리 타이틀**: "ROLLING THUNDER" 타이포 아래로 네온 칩들이 실제 Rapier.js 물리로 낙하·충돌하는 캔버스 배경. 마우스 포인터가 보이지 않는 강체(collider)가 되어 칩을 밀어낼 수 있다 — *방문자가 첫 3초 안에 물리 엔진을 손으로 만진다.*
2. **라이브 3초 데모 입력창**: 히어로 중앙에 인라인 입력창 하나 — "친구들 이름을 쉼표로 넣어보세요". 입력 즉시 이름표가 붙은 칩이 그 자리에서 굴러떨어지는 미니 추첨 프리뷰. CTA로 자연 연결: **"진짜 레이스는 대기실에서"**
3. **카피 후보**:
   - 헤드라인: `사다리타기는 이제 그만. 중력에게 맡기세요.`
   - 서브: `Rust 물리 엔진(Rapier) 위에서 펼쳐지는 도파민 터지는 실시간 추첨 레이스 — 설치 없이, 브라우저에서 바로.`
4. **CTA 3종**: `무료로 시작하기`(회원가입 모달) · `설치 없이 바로 체험`(게스트 → /dashboard) · `로그인`(헤더 우측)
5. **성능/모션 안전장치**:
   - 물리 캔버스는 `requestIdleCallback` 이후 lazy-init (WASM ~수백 KB — LCP 블로킹 금지)
   - `prefers-reduced-motion` 또는 calmMode Tier1 감지 시 → 정적 그라디언트 + 미세 패럴랙스로 대체
   - 모바일 저사양: 칩 수 상한(예: 24개) + 30fps 캡

카드 그리드(2×3, 모바일 1열). **전부 실제 존재하는 기능만** 기재한다:

| 카드 | 소구 포인트 (근거 코드) |

## 5. 로그인/회원가입 모달 (Headless UI)

기존 `page.tsx`의 폼 로직·검증 규칙을 **그대로 이식**하고 껍데기만 교체한다.

| 요소 | Headless UI 컴포넌트 | 비고 |
|------|------|------|
| 모달 컨테이너 | `Dialog` + `DialogPanel` + `DialogBackdrop` | `transition` prop + Tailwind `data-closed:opacity-0` 패턴 (v2 방식) |
| 로그인 ↔ 회원가입 전환 | `TabGroup`/`TabList`/`TabPanel` | 기존 `isLogin` state 토글 대체 |
| 입력 필드 | `Field` + `Label` + `Input` + `Description` | 기존 검증 규칙 유지: 아이디 `/^[a-zA-Z0-9_]{3,20}$/`, 닉네임 ≤ 10자, 비밀번호 확인 일치 |
| 로그인 상태 유지 | `Checkbox` | 기존 `keepLoggedIn` 쿠키 로직 그대로 |
| **약관 동의 (신규)** | `Checkbox` ×2 (필수) | ① "[이용약관](/terms)에 동의합니다" ② "[개인정보처리방침](/privacy)을 확인했으며 동의합니다" — 링크는 새 탭. 미체크 시 가입 버튼 비활성 |
| 에러 표시 | 기존 경고 박스 스타일 유지 | `aria-live="polite"` 부여 |

**접근성 요구**: 포커스 트랩(Headless UI 기본 제공), `Esc` 닫기, 모달 오픈 시 배경 스크롤 잠금, 첫 필드 자동 포커스.

**서버 액션 변경(최소)**: `signup()`에 `termsAgreed` 검증 1줄 추가(미동의 시 에러 반환). 동의 시각의 DB 기록은 v2.

## 6. 서비스 이용약관 (`/terms`) — 내용 요구사항

약관의 규제에 관한 법률(약관규제법) 및 표준 온라인 서비스 약관 구조를 따르되, **모든 조항이 이 앱의 실제 동작을 서술**해야 한다. 조항별 필수 내용:

| 조 | 제목 | 반드시 담을 실제 구현 사실 |
|----|------|------|
| 1 | 목적 | 서비스명 "Rolling Thunder"(물리 엔진 기반 무작위 추첨 웹 서비스), 제공자 "찰떡쌤" |
| 2 | 정의 | 회원/비회원(게스트), 칩(무료 게임 재화), 스킨, 커스텀 맵, 대기실, 세션 |
| 3 | 약관의 게시와 개정 | 랜딩 푸터 상시 게시. 불리한 개정은 **30일 전 공지**, 시행일·개정사유 명시 |
| 4 | 회원가입 | 아이디(영문·숫자·밑줄 3~20자)+비밀번호+이름+닉네임(≤10자)으로 가입. **이메일 인증 없음**(내부적으로 가상 주소 체계 사용) 명시. 타인 명의·정보 도용 금지 |
| 5 | 서비스의 내용 | 추첨 레이스, 맵 에디터, 커스텀 맵 스토어, 가챠/상점, 미션·업적. **무료 제공**이며 광고·유료 결제 없음(현행 기준) |
| 6 | 게스트 이용 | 비회원 데이터는 브라우저(localStorage)에만 저장되어 **기기/브라우저 삭제 시 소실**될 수 있음. 회원 전환 시 칩·인벤토리 승계됨 |
| 7 | 칩·가챠·가상 재화 | 칩은 **현금 구매 불가·환불 대상 아님**(무상 지급 재화). 확률형 아이템(가챠)의 결과는 무작위이며 재화 가치의 현금 전환 불가. 회원 탈퇴 시 소멸 |
| 8 | 이용자 제작 콘텐츠(UGC) | 커스텀 맵·참가자 명단의 권리는 제작자에게 있으나, 스토어 공개 시 서비스 내 노출·복제(다운로드)에 필요한 이용권을 서비스에 허여. 타인 저작권/상표 침해 콘텐츠, 혐오·모욕성 맵 이름 금지 및 관리자 삭제 권한 |
| 9 | 금지행위 | 해킹/비인가 API 호출, 매크로, 칩·통계 조작, 타인 계정 도용, 서비스 역설계 후 무단 재배포 |
| 10 | 서비스의 중단·변경 | 무료 서비스 특성상 사전 공지 후 변경·중단 가능. 정기 점검, Supabase/Vercel 등 인프라 장애 시 면책 |
| 11 | 책임의 한계 | **추첨 결과의 공정성은 물리 시뮬레이션에 기반하나, 상금·경품 등 외부 보상의 분배 책임은 이용자에게 있음**(중요 — 굵게 표기). 무료 서비스로서 데이터 유실에 대한 배상 한계 |
| 12 | 계약 해지(탈퇴) | 프로필 메뉴에서 즉시 탈퇴 가능, 탈퇴 시 계정·게임 데이터 **즉시 완전 삭제**(복구 불가) |
| 13 | 준거법·분쟁 | 대한민국 법. 분쟁 시 상호 협의 우선 |

## 7. 개인정보처리방침 (`/privacy`) — 내용 요구사항

개인정보 보호법 **제30조 필수 기재사항**을 전부 커버하되, 항목 하나하나를 실제 스키마·코드와 대조해 작성한다.

### 7.1 수집 항목 (실제 스키마 기준 — 이 표를 방침 본문에 그대로 수록)

| 구분 | 항목 | 근거 (코드/테이블) | 보유 기간 |
|------|------|------|------|
| 필수 (가입 시) | 아이디, 비밀번호(해시 저장), 이름, 닉네임 | `profiles(username, name, nickname)`, Supabase Auth(bcrypt) | 회원 탈퇴 시 즉시 파기 |
| 자동 생성 | 가입일시, 권한 등급(role) | `profiles(created_at, role)` | 탈퇴 시 즉시 |
| 서비스 이용 | 칩 잔액·변동 이력, 가챠 기록, 인벤토리/장착 스킨, 미션·업적·스탬프, 제작 맵·좋아요·다운로드 이력, 게임 세션·결과 | `chip_logs`, `gacha_logs`, `user_inventory`, `user_missions`, `user_achievements`, `user_maps`, `user_map_likes/downloads`, `sessions/participants/results` | 탈퇴 시 즉시 (FK CASCADE) |
| **이용자가 입력하는 제3자 정보** | 참가자 명단(추첨 대상자 이름/닉네임) | `participant_lists`, `participants` | 탈퇴 또는 명단 삭제 시 |
| 쿠키 | Supabase 인증 토큰(`sb-*`), 로그인 유지 설정(`keep_logged_in`, 최대 1년) | `middleware.ts` | 로그아웃/만료 시 |
| 비회원(게스트) | 칩·인벤토리·장착 정보 — **서버 미전송, 브라우저 localStorage에만 저장** | `actions.ts` 게스트 승계 로직 | 이용자가 브라우저에서 직접 삭제 |

> ⚠️ **참가자 명단 특칙(차별점)**: 이 서비스는 이용자가 *타인의 이름*(학생·동료 등)을 입력하는 구조다. 방침에 "참가자 명단은 추첨 진행 목적으로만 저장되며, 명단 속 개인의 삭제 요청은 명단 소유 회원 또는 문의 이메일로 처리한다"는 조항을 반드시 포함할 것. 실명 대신 별명 사용 권장 문구도 안내.

## 8. 디자인 & 기술 요구사항

### 디자인

- 기존 네온 다크 테마 CSS 변수 그대로 사용 (`--bg-primary` `#101217`, `--accent-primary`, `glass-panel-heavy`) — 랜딩과 앱의 시각 연속성 유지
- 타이포: 헤드라인 Outfit(`--font-outfit`), 본문 Pretendard. 법적 문서는 가독성 우선 — `max-w-3xl`, `leading-relaxed`, 조항 번호 목차
- 반응형: 360px(모바일) ~ 1920px. 히어로 캔버스는 `100dvh` 기준, 랜딩 페이지는 `overflow-y-auto` 필요(현재 `body`가 `overflow-hidden`이므로 **랜딩 전용 스크롤 컨테이너** 처리 주의)
- 모션: framer-motion 스크롤 리빌(`whileInView`), stagger 카드 등장. **calmMode Tier1 + `prefers-reduced-motion` 시 전부 fade-only로 강등**

### 기술

- **구현 착수 전 `node_modules/next/dist/docs/` 필독** (AGENTS.md 지침 — 이 프로젝트의 Next.js는 관례가 다를 수 있음)
- 랜딩 셸·법적 문서 = 서버 컴포넌트(정적 렌더). 물리 히어로·로그인 모달만 `'use client'` 아일랜드로 분리
- 법적 문서 본문은 TSX 내 콘텐츠 상수 또는 컴포넌트로 관리(마크다운 파서 추가 의존성 불필요), 최종수정일·버전 상수 export
- 메타데이터: `/` 전용 `metadata`(title/description/OG 이미지), `/terms`·`/privacy` 개별 title. JSON-LD `SoftwareApplication` 스키마(선택)
- 성능 예산: LCP < 2.5s(히어로 텍스트가 LCP가 되도록 — 캔버스는 지연 로드), 물리 WASM은 상호작용 전 프리로드 금지
- 사운드: 랜딩에서는 BGM 자동재생 금지(기존 `soundManager.stopAllBgm()` 동작 유지)
